# Soloship Retro

Invoke the `retro` skill (gstack). It runs a weekly retrospective analyzing:

- Commit history and work patterns
- Code quality metrics and trends
- What went well, what didn't
- Per-contributor breakdown (if team project)

## When to Run

Run weekly, or after completing a major milestone. Helps catch patterns
before they become problems — increasing complexity, decreasing test coverage,
mounting tech debt.

## Verification

Retro is not complete until ALL of these are true:

- [ ] Commit history analyzed (not just summarized — patterns identified)
- [ ] What went well AND what didn't both addressed
- [ ] Actionable recommendations included (not just observations)
- [ ] Retrospective delivered to user
